# Feature 模板

交付周期：1~4 周（内部发布或灰度发布）

Feature 是为用户提供完整价值的最小应用（功能集）。

**示例：** 登录注册、销售数据统计报表、工单数据导入导出。

Feature 是相关的功能需求的集合，对于更具体的用户场景，则通过 User Story 来细分和描述，所以 Feature 可视为是一组故事的集合，包含用户故事和技术故事。

```
Epic ──► Feature ──► User Stories
```

## 特性模板

### 需求场景 & 用户痛点

**责任人：** PO

描述用户的使用场景，在该场景下的痛点/诉求，有什么问题。

### 需求洞察 & 特性设计

**责任人：** PO

1. **洞察用户核心需求，规划特性，解决用户痛点**
2. **FAB 方法：**
   - **Feature（功能特性）**：是什么功能
   - **Advantages（有什么作用，解决什么问题）**：解决什么问题
   - **Benefit（给用户带来什么价值、利益）**：带来什么价值

### 特性价值度量指标

**责任人：** PO

衡量特性价值的定量/定性指标。

| 类型 | 指标描述 |
|-----|---------|
| 定量指标 | |
| 定性指标 | |

### 依赖与风险

**责任人：** PO & TL

| 类型 | 描述 |
|-----|------|
| **需求关联性** | 内部功能依赖，外部跨团队协作 |
| **技术风险** | 技术风险、测试风险 |

### 技术思路/工作量

**责任人：** TL

- 预研项识别
- 提供技术思路或技术方案
