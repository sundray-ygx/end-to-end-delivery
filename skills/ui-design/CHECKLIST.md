# UI/UX 预交付质量检查清单

本文档提供完整的UI/UX交付前质量检查清单，整合了三大插件的所有检查项。

## 检查清单概述

在交付UI代码之前，按以下类别逐一验证：

### A. 视觉质量 (Visual Quality)

| 检查项 | 说明 | 验证 |
|--------|------|------|
| **无Emoji图标** | 使用SVG图标（Heroicons/Lucide），不用emoji作为UI图标 | [ ] |
| **一致图标集** | 所有图标来自同一图标集，保持视觉一致性 | [ ] |
| **正确品牌Logo** | 从Simple Icons验证官方SVG路径 | [ ] |
| **稳定悬停状态** | 悬停不使用导致布局偏移的scale变换 | [ ] |
| **直接使用颜色** | 使用`bg-primary`而非`var(--color-primary)`包装 | [ ] |

### B. 交互 (Interaction)

| 检查项 | 说明 | 验证 |
|--------|------|------|
| **光标Pointer** | 所有可点击元素添加`cursor-pointer` | [ ] |
| **悬停反馈** | 悬停状态提供清晰视觉反馈（颜色/阴影/边框） | [ ] |
| **平滑过渡** | 过渡时间150-300ms，使用缓动函数 | [ ] |
| **焦点状态** | 键盘导航时焦点状态可见 | [ ] |
| **加载状态** | 异步操作时按钮禁用且有加载指示 | [ ] |

### C. 亮/暗模式 (Light/Dark Mode)

| 检查项 | 说明 | 验证 |
|--------|------|------|
| **亮模式对比度** | 文本对比度≥4.5:1 | [ ] |
| **暗模式对比度** | 文本对比度≥4.5:1 | [ ] |
| **玻璃卡片可见性** | 亮模式使用`bg-white/80`+，暗模式`bg-black/50`+ | [ ] |
| **边框可见性** | 两种模式下边框都可见 | [ ] |
| **测试两种模式** | 交付前测试亮色和暗色主题 | [ ] |

### D. 布局 (Layout)

| 检查项 | 说明 | 验证 |
|--------|------|------|
| **浮动元素间距** | 浮动导航栏等与边缘有适当间距 | [ ] |
| **内容可见性** | 没有内容隐藏在固定导航栏后面 | [ ] |
| **响应式断点** | 测试375px、768px、1024px、1440px | [ ] |
| **无水平滚动** | 移动端无意外水平滚动 | [ ] |
| **Z-index管理** | 使用预定义z-index值（10, 20, 30, 50） | [ ] |

### E. 无障碍 (Accessibility)

| 检查项 | 说明 | 验证 |
|--------|------|------|
| **Alt文本** | 所有有意义图片有描述性alt文本 | [ ] |
| **表单标签** | 输入框有关联的label（for属性） | [ ] |
| **非颜色指示** | 颜色不是唯一的信息指示器 | [ ] |
| **减少动效** | 尊重`prefers-reduced-motion` | [ ] |
| **ARIA标签** | 纯图标按钮有aria-label | [ ] |
| **键盘导航** | Tab顺序与视觉顺序一致 | [ ] |

### F. 性能 (Performance)

| 检查项 | 说明 | 验证 |
|--------|------|------|
| **图片优化** | 使用WebP、srcset、lazy loading | [ ] |
| **内容预留** | 异步内容加载前预留空间 | [ ] |
| **动画性能** | 使用transform/opacity，不用width/height | [ ] |
| **字体加载** | 使用font-display: swap避免FOUT | [ ] |

### G. 字体与排版 (Typography)

| 检查项 | 说明 | 验证 |
|--------|------|------|
| **行高** | 正文1.5-1.75，标题1.1-1.3 | [ ] |
| **行长度** | 每行≤75字符 | [ ] |
| **字体大小** | 移动端正文≥16px | [ ] |
| **字体搭配** | 标题/正文字体性格匹配 | [ ] |

### H. 色彩 (Color)

| 检查项 | 说明 | 验证 |
|--------|------|------|
| **对比度检查** | 所有文本与背景对比度≥4.5:1 | [ ] |
| **CSS变量** | 使用CSS变量定义颜色系统 | [ ] |
| **色彩一致性** | 颜色使用与设计系统一致 | [ ] |
| **避免陈词滥调** | 不使用紫色渐变等常见AI模式 | [ ] |

### I. 动效 (Animation)

| 检查项 | 说明 | 验证 |
|--------|------|------|
| **持续时间** | 微交互150-300ms | [ ] |
| **缓动函数** | 使用ease-out或自定义缓动 | [ ] |
| **加载状态** | 骨架屏或加载指示器 | [ ] |
| **减少动效** | 检查prefers-reduced-motion | [ ] |

### J. 代码质量 (Code Quality)

| 检查项 | 说明 | 验证 |
|--------|------|------|
| **语义化HTML** | 使用正确的HTML5元素 | [ ] |
| **组件结构** | 组件有清晰的职责划分 | [ ] |
| **响应式设计** | 移动优先，渐进增强 | [ ] |
| **浏览器兼容** | 测试主流浏览器 | [ ] |

## 专业UI额外检查

### 图标与视觉

```
[ ] 不使用emoji作为功能图标
[ ] 使用Heroicons、Lucide或Simple Icons
[ ] 图标使用固定viewBox（如24x24）
[ ] 图标尺寸一致（如w-6 h-6）
```

### 交互反馈

```
[ ] 所有可点击元素有cursor-pointer
[ ] 悬停状态有视觉变化
[ ] 过渡平滑，不突兀
[ ] 焦点状态清晰可见
```

### 常见错误检查

```
[ ] 没有使用Inter/Roboto/Arial等通用字体
[ ] 没有使用紫色渐变等陈词滥调
[ ] 没有scale导致布局偏移
[ ] 没有低对比度文本
[ ] 没有不可见边框（亮模式border-white/10）
```

## 快速检查命令

### 使用Web Interface Guidelines审查

```bash
# 检查特定文件
curl https://raw.githubusercontent.com/vercel-labs/web-interface-guidelines/main/command.md | bash

# 检查整个项目
npx @vercel-labs/web-interface-guidelines check .
```

### 手动检查要点

```
1. 打开开发者工具
2. 检查Console无错误
3. 检查Network无404
4. 检查Lighthouse得分
   - Performance ≥ 90
   - Accessibility ≥ 90
   - Best Practices ≥ 90
5. 检查响应式（开发者工具设备模拟）
```

## 常见问题自查

| 问题 | 检查 | 解决 |
|------|------|------|
| **布局偏移** | CLS分数 | 为图片和异步内容预留空间 |
| **低对比度** | 对比度检查工具 | 调整颜色 |
| **不可点击** | cursor检查 | 添加cursor-pointer |
| **悬停无反应** | CSS检查 | 添加hover状态 |
| **键盘无法访问** | Tab导航测试 | 确保Tab顺序正确 |
| **移动端水平滚动** | 视口宽度检查 | 确保内容适合视口 |
